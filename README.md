# UX Interaction Audit

中文 UX / 交互合理性审查 Codex Skill。

这个 Skill 用于评估界面截图、设计稿、App/Web 页面或页面描述，帮助快速判断页面交互是否合理，并输出可执行的修改建议。它适合用于产品评审、设计走查、表单/流程优化、转化页诊断和生成前后对比标注图。

## 能做什么

- 识别页面目标、目标用户、核心任务和主行动
- 还原用户完成任务的路径
- 输出最重要的 5 个 UX / 交互问题
- 按 9 大 UX 层面评分
- 给出修改规则清单，说明改什么、保留什么、禁止什么
- 输出文案替换建议
- 在需要出图时，生成前后对比标注看板的提示与规则

## 默认输出结构

默认输出重点版 UX 审查文档，避免信息过载：

1. 页面基础判断
2. 用户任务路径分析
3. 最重要的 5 个问题
4. 9 大 UX 层面评分
5. 修改规则清单
6. 文案替换建议
7. 一句话总结

默认不展开完整 9 层逐项审查、优先级排序、完整自检表、参考来源 URL 和设计规范表。需要时可以明确要求展开。

## 9 大 UX 层面

- 视觉层面
- 信息层面
- 交互层面
- 流程导航层面
- 反馈状态层面
- 表单与错误层面
- 无障碍层面
- 信任与风险层面
- 情绪体验层面

每个关键问题会绑定参考原则和来源类型，避免把启发式原则、理论来源和官方标准混写。

## 图片产物规则

默认只输出文字审查和标注点清单，不主动生成图片。

只有用户明确要求以下内容时，才生成图片产物：

- 生成对比图
- 输出图片产物
- 修改后示意图
- 前后对比看板
- 给我图

生成优化后页面或前后对比图前，Skill 会先在内部从上传图片中提取设计规范，并把规范应用到修改后页面。默认不输出色彩、字体、间距、组件等设计规范表，除非用户明确要求展示。

右侧修改后页面应像原图的同设计系统升级版，而不是另一套新 UI。

## 推荐调用方式

```text
用 UX Interaction Audit 评估这张图
```

```text
用 UX Interaction Audit 评估这张图，并生成一张前后对比标注看板：左侧放修改前原图，右侧放修改后建议示意图，中间放 6-10 条编号改动清单，每条包含问题标题和一句修改建议，箭头指向修改后对应区域。截图保持等比，不拉伸、不裁切。
```

```text
用 UX Interaction Audit 做完整审查，并展开 9 大 UX 层面、完整自检和参考来源 URL。
```

## 安装

将本仓库作为 Skill 放入 Codex skills 目录：

```bash
mkdir -p ~/.codex/skills
git clone https://github.com/SemineChen/ux-interaction-audit.git ~/.codex/skills/ux-interaction-audit
```

如果已经安装过，更新即可：

cd ~/.codex/skills/ux-interaction-audit
git pull

## 适用场景

- App 页面体验评估
- Web 页面体验评估
- 表单和提交流程审查
- 注册、报名、支付、授权等关键流程诊断
- 设计稿改版建议
- 图片标注和前后对比看板生成

## 输出原则

这个 Skill 的重点不是判断页面“好不好看”，而是判断用户能否快速理解页面、顺利完成任务，并知道每一步操作会产生什么结果。

修改建议会尽量保持原页面的业务目标和设计系统，只优化信息层级、交互状态、反馈、文案和任务路径。
